# Export Dialog

> Component: Modal Dialog · Triggered from: Status Bar `[Export]` button, `Ctrl/Cmd+E`

---

## Overview

The Export dialog lets the user export the current screen (or all screens in the project) in one of
three formats. Each format targets a different downstream consumer.

---

## Layout

```
┌──────────────────────────────────────────────────┐
│  Export Screen                              [✕]  │
│  ──────────────────────────────────────────────  │
│  [SDUI JSON]  [Dart Widget]  [Flutter Project]   │
│  ────────────────────────────────────────────── │
│                                                  │
│  <tab content — see below>                       │
│                                                  │
│  ────────────────────────────────────────────── │
│  [📋 Copy to Clipboard]          [⬇ Download]   │
└──────────────────────────────────────────────────┘
```

---

## Tab 1 — SDUI JSON

**Purpose:** Export the raw SDUI template JSON for use with `zugantor_sdui` at runtime.

**Content shown:**

```json
{
  "screen_id": "profile_screen",
  "schema_version": "1",
  "layout": { ... }
}
```

**Options:**
- **Scope:** Current screen (default) or All screens (separate files in a zip)
- **Format:** Minified or Pretty-printed (toggle, default: pretty)

**Copy:** Copies the JSON string to clipboard.  
**Download:** Saves as `<screen_id>.sdui.json` (or `<project_name>_screens.zip` for all screens).

---

## Tab 2 — Dart Widget

**Purpose:** Export the screen as a `StatelessWidget` that can be pasted into any Flutter project.

**Content shown:**

```dart
// Auto-generated by Zugantor Build
// Screen: profile_screen
import 'package:flutter/material.dart';

class ProfileScreen extends StatelessWidget {
  const ProfileScreen({super.key});

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        // ...
      ],
    );
  }
}
```

**Notes:**
- ZDS imports use the actual `zugantor_design_system` package import path.
- Bind expressions (`$.user.name`) are converted to constructor parameters with defaults.
- The generated widget is `dart format`-compliant.

**Copy:** Copies the Dart source code to clipboard.  
**Download:** Saves as `<screen_id_pascal_case>.dart`.

---

## Tab 3 — Flutter Project

**Purpose:** Export a full standalone Flutter project containing this project's screens as a ready-to-run app.

**Content shown:**
- File tree preview: `lib/main.dart`, `lib/screens/*.dart`, `pubspec.yaml`, etc.
- Read-only, for user reference.

**Options:**
- App name field (used as package name and display name)
- Target Flutter SDK version (dropdown, default: latest stable)

**Download only (no clipboard):** Saves as `<project_name>.zip`.

**Zip contents:**
```
<project_name>/
  pubspec.yaml
  analysis_options.yaml
  lib/
    main.dart
    screens/
      <screen_id>.dart
      ...
  assets/
    ...
```

---

## Validation Before Export

- Template JSON is schema-validated before any export.
- If validation fails, a warning banner is shown inside the dialog:
  > ⚠ This screen has validation errors. Export may produce an invalid template.
  
  The user can still export (override), but it is flagged.

---

## Actions

| Element | Behaviour |
|---|---|
| Tab selection | Switches export format preview |
| `[📋 Copy to Clipboard]` | Copies current tab's output to OS clipboard; shows "Copied!" toast |
| `[⬇ Download]` | Triggers browser/OS save dialog with pre-filled filename |
| `[✕]` / outside click | Closes dialog, no file written |
